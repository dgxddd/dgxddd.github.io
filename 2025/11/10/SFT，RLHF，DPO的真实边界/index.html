SFT（监督微调）

是使用标注的训练数据对预训练模型进行微调，使其在特定任务上性能更好。模型学习大量「输入 - 标准输出」标注对，通过自回归对数似然损失反向传播，迭代更新模型参数 θ，以便在给定输入下输出期望的结果。这种方法简单直接，适用于有大量高质量标注数据的任务。

RLHF

RLHF的训练过程可以分解为三个核心步骤：

步骤一：收集示范数据，训练监督策略 从提示词数据集采样提示词 → 标注人员给出符合预期的理想输出 → 用（指令-答案）数据监督微调GPT-3 (这个用的SFT)

步骤二：收集对比数据，训练奖励模型 对同一提示词，指令微调模型生成多个回复 → 标注人员对回复从优到劣排序 → 用偏好对比数据训练奖励模型RM （训练奖励模型 RM）

步骤三：基于奖励模型，强化学习优化策略 从数据集采样新提示词 → 策略模型生成回复 → 奖励模型打分 → 用PPO算法更新策略模型权重

分数越高 = 越符合人类喜好、越优质；
分数越低 = 废话、跑偏、违规、人类不喜欢。

PPO

其中策略模型由训练好的 SFT 模型初始化而来，全程参与训练、参数一直在变。

参考模型是原始 SFT 模型完整复制一份，全程冻结、参数永远不动

GAE 就是用评论家模型的基线，把「当前奖励 + 未来多步奖励」揉在一起平滑计算，算出又准又稳的优势函数 A_t，专治 PPO 训练震荡、不稳定。

其中PPO-clip裁剪损失计算。

其中r
r_t(θ)：新策略 / 旧策略 生成同一个 token 的概率比值，代表更新幅度。

A_t：GAE 算出来的优势函数。其中A_t>0（回答比基线好）：鼓励模型多沿用这种生成方式，但限制更新幅度，不能学太疯。

A_t<0 (回答比基线差）：抑制这种生成方式，同样控制修改力度

ε：是超参数，用于控制裁剪幅度。

DPO

第一步

  • 输入提示：就是我们给模型的问题/指令；
  • 优选高质量回复：人类认可、符合需求的好回答；
  • 劣选低质量回复：人类不认可、不符合需求的坏回答

第二步

  • 参考模型（SFT模型）：就是我们提前训好的“基础版模型”，训练时固定它的参数（不改动），当作“基准线”；
  • 策略模型：就是我们要最终训练好的“对齐版模型”，只有它的参数会被更新

第三步

  • “最大化优选相对劣选的对数概率比值”：逼着模型让“好回答（Chosen）”的生成概率变高，让“坏回答（Rejected）”的生成概率变低，相当于直接告诉模型“选这个、不选那个”；
  • “隐式等价学习奖励函数”：不用RM给回答打分，DPO通过这种“好坏对比”，会自动学会“什么是好、什么是坏”，相当于把“奖励模型的功能”藏在了对比训练里；

第四步

  • KL散度约束：简单说就是“衡量两个模型（策略模型和参考模型）输出差异的指标”；
  • “控制偏移”：通过KL约束，让策略模型（要训的模型）的输出，不能和参考模型（基础模型）差太远

第五步

  • 通过前面的“好坏对比”和“KL约束”，计算出模型参数需要调整的方向（梯度），逐步更新策略模型的参数

  • ：就是DPO的损失值，训练的目标是“让这个损失值越小越好”，损失越小，模型越符合人类偏好；
  • ：策略模型（我们要训的模型）在输入提示x下，生成好回答（y_w，即Chosen）的概率；
  • ：参考模型（冻结的SFT模型）在输入提示x下，生成好回答（y_w）的概率；
  • ：策略模型在输入提示x下，生成坏回答（y_l，即Rejected）的概率；
  • ：参考模型在输入提示x下，生成坏回答（y_l）的概率；
  • ：就是前面提到的KL权重（默认0.1），核心作用是控制策略模型和参考模型的偏离程度，平衡“对齐偏好”和“避免崩坏”；
  • ：Sigmoid函数，简单说就是“把计算结果压缩到0-1之间”，方便计算损失、判断模型的偏好学习效果。

核心逻辑（对应第三步流程）：

损失函数的核心是“让好回答的相对概率更高，坏回答的相对概率更低”——公式里的两个对数比值，其实就是在计算“策略模型比参考模型，更愿意生成好回答、更不愿意生成坏回答的程度”，再通过控制这种“偏好倾向”不太过火（避免偏离参考模型太远），最后通过损失值的降低，逐步调整策略模型的参数
