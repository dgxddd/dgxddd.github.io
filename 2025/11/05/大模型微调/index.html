大模型微调

首先我们思考一个问题为啥要有大模型微调？在回答这个问题之前我们先去思考大模型是如何训练更新的，是为啥可以像一个“老师”一样啥都懂。

首先大模型会经历预训练，而在预训练中会经历一下几个阶段

1.数据准备：这个时候会给大模型准备好成千上万的知识来帮助其学习

2.模型初始化：随机初始化模型的所有参数

3.自监督学习：这是预训练最核心的机制。模型通过 “预测下一个 token” 的任务来学习

而在这个过程中大模型的预训练就会出现问题比如虽然大模型学习了许多知识但是这个时候呢他还是会有能力边界，这个大家应该有体会在我们问大模型的时候会胡乱说，还有对于某个专业的知识不足等。

这个时候我们就可以回答了上一个问题，微调的本质，就是在预训练模型的基础上，使用特定任务的数据集进行进一步训练，让其更专业

当然在让大模型适应特定任务还有提示词工程，还有RAG这个我们不在这里讲。

为啥要微调呢，这个时候我们从训练需要的显存开始说，

1.加载模型权重 → 2. 前向传播（算激活值）→ 3. 算损失 → 4. 反向传播（算梯度）→ 5. 优化器更新权重

其中模型权重就是预训练大模型的全部可学习参数。

激活值就是前向传播时，模型每一层的中间输出结果。你可以理解这个是为了来计算梯度的

大模型微调之全量微调

首先我们先讲解一下全量微调，这个就是意思我们在每次更新预训练模型的所有参数，虽然这样效果最好但是太消耗显存了，这也是我们成本大幅度增加。 这个时候我们就想到了LORA，让其训练量少了很多。

大模型微调之LORA

既然全量微调成本太高，那有没有一种方法，既能让模型适配特定任务，又能节省资源？当然有——LORA，全称 Low-Rank Adaptation（低秩适应），就是为解决全量微调的痛点而生的。这个是通过调节模型参数来降低显存的使用。

我们可以将其看成是矩阵的乘法,本质上如下图所示

LoRA 不是改动原有大模型权重，而是额外插入两个低秩小矩阵，维度做成 n×k,k×m用这两个小矩阵去近似模拟原大矩阵的更新量，
原模型全部参数直接冻结不动。

其中这个r就是我们中间的k，即设置的小矩阵中间维度，α：缩放超参数，一般初始设成和 r 一样大。

前向传播：原输出 + AB 乘积融合，Y=XW+X⋅AB⋅ a/r，这样我们就只需要nk+km的参数量 ，就能以极低资源消耗，逼近全量微调的任务适配效果

大模型微调之QLORA

lora虽然降低了更新的参数但是其原来本身的参数没有降低，基座模型还是 16-bit 全精度存储，为了解决这个问题，QLoRA 才应运而生。它在 LoRA 的基础上，把基座模型用 4-bit 量化存储，配合 NF4 量化、双重量化和分页优化器技术，大幅度的降低了显存。对比图如下所示。

其中主要来解决3个问题，第一个是基础模型16bit改为4bit。第二个是量化后缩放因子占用额外显存，这个时候我们进行二次量化。第三个是微调时 Adam 优化器会太多我们使用分页优化器把优化器状态分页存到 CPU 内存，GPU 按需加载。

4-bit NF4 量化的处理

4-bit NF4量化是QLoRA的基础核心技术，其核心目标是将16-bit全精度基座模型压缩为4-bit存储，同时最大限度减少精度损失。

二重量化

在完成4-bit NF4量化后，虽然权重显存占用降低至原来的1/4，但量化过程中产生的缩放因子scale会带来额外的显存开销，这也是QLoRA需要解决的第二个核心问题，具体细节结合下图说明：
